Idea

When I read the requirements of the email and the fact that there was a limit to how big the file could be I instantly thought about making an API server. So I learnt how to make a mini API with flask and python so I made this. Now since this is going out to the internet , I feel it would be a bad idea to just leave my data in plain English so therefore I encoded everything into base64 encryption. This will both secure my ‘data’ and make it seem like gibberish to someone who could accidentally just visit the API site.

I made an API with Flask , this is good because A) I can now make my supporting statement as long as I want and B) I can now use some sort of encryption adding to the security of my API.

The problem

The problem I came across was that, sure I can make another python script with requests and then just request and .text() all the information I need to GET and print it to your screen BUT what if you don’t have python installed?

Therefore, my solution was batch scripting. My knowledge on batch is not much but with research I learned about curl (GET the data) and cert util(Decode the data). This is perfect because now I can both get the necessary information from my API as well as decode the Base64 encryption. Hopefully while I make this project I can teach myself on how to assign variables and possibly get my head around IF, WHILE and FOR loops with batch scripting. This limits my chance to show my “finest skills” however I will attempt to learn and use these statements.
